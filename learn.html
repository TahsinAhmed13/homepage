Getting Started

*nix

To open the terminal press Crtl+Alt+T on your keyboard. Because this might accidentially open the terminal on your own computer for this site you can use the T key to open the terminal at any time.

Windows

Windows does not come with bash as the default terminal program. Therefore to follow along on this guide you will have to install it, which can be done by installing a terminal program like Cygwin.

Commands

The terminal does not have a GUI, so you will have to use text commands to get around. You may be asking well why would I use this then? To that I can assure you that the terminal is more powerful than any GUI program in existence. Here is a list of the most useful commands for beginners.

Note: the terminal emulator on this website isn't that robust and supports only the first four commands

  1. pwd

    > pwd

    Function: prints the current working directory

  2. ls

    > ls

    Function: lists all the files and directories under the current directory

    Tip: in blue are directories and in white are files

  3. cd

    > cd [directory]

    Args: the directory you want to connect to

    Function: connects to a given directory

    Tip: to get a file/dir deep inside the current dir use '/' between the dirs that your going through

    Tip: '.' is the current directory and '..' is the parent directory

  4. cat

    > cat [files...]

    Args: can be up to any number of file names

    Function: concatanates (joins) the contenets of all the files, but is commonly used to see the contents of one file

  5. mkdir

    > mkdir [name]

    Args: the name of the directory

    Function: makes a new directory in the current one

  6. rmdir

    > rmdir [name]

    Args: the name of the directory

    Function: removes a directory

    Tip: this only works on empty dirs, otherwise use rm -rf [name]

  7. touch

    > touch [name]

    Args: the name of the file

    Function: creates a zero byte file

  8. rm

    > rm [name]

    Args: the name of the file

    Function: removes a file

  9. mv

    > mv [from] [to]

    Args: where the file is currently located

    Args: path where the file should go

    Function: moves a file

    Tip: mv can also be used to rename if for the destation a new name is appended at the end

  10. cp

    > cp [original] [new]

    Args: where the file is current located

    Args: path where it should be copied

    Function: copies a file from one location to another

    Tip: the name can also be changed if at the end an alternate name is appended

Note: if you use tab after typing some portion of an args it will auto-complete

Note: pressing the UP key will fetch the last called command
